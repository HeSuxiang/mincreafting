製作Mods

該頁面的內容不受Mojang AB、Minecraft Wiki、Minecraft IRC頻道與Minecraft論壇的官方支持。

ModLoader或者直接修改源代碼的方法不再被推薦；Forge(或liteloader或其他的加載器)更受歡迎。另外，可能很多信息只是在使用過時的技術；本文需要進行一次編輯，以確保所有內容都是最新的。

Minecraft Mod製作是這個社區中最重要的功能之一。世上有很多種類的Mod，這個教程將會給你提供製作你自己的Mod的信息無論它將基於ModLoader或直接在核心代碼中編寫。一些Java的編程知識是必要的，這個教程將假設讀者已經熟悉Java編程語言的基礎語法。你可以為所有的計算機運作系統(Windows、OS X、Linux、Android、IOS、Raspberry Pi和Windows Phone)創建mod。你不能在控制台製作mod。你可以使用本頁面提供的工具和知識為Minecraft製作mod，還可以改進使用的Java甚至是一些進展中的程序語言。希望該過程一路順利。

工具

許多工具都可以開發Minecraft的mod。下面的工具列表是人們放置的或在討論頁中推薦的。Minecraft通常是使用的Java語言，所以為Minecraft製作mod時幾乎全部都是在編譯Java。同時Java 9快照不能用於Minecraft，但是Java 8快照可以，如果你需要運行最新的Java 8快照，下面的這些Java都是可以使用的：

  • Java SE(Java標準版)，能夠用於下列包：Java的正常運行環境，JRE(Java運行環境)和JDK(Java開發工具)。JDK是為Java開發者和編譯者設計的，需要用來編譯Minecraft的mod。
  • Java EE(Java企業版)，用於商業、公司和大型項目，與標準版Java相比，有更多的類。
  • Java ME(Java微型版)是Java的小型版本。
  • JavaFX，現在已經被包括在所有的Java包。
  • 嵌入Java，對於嵌入的設備(是，你可以在一些有Windows、OS X或Linux操作系統的系統運行Minecraft)。

任何設備都通用的Java編碼工具可以用來幫助創建mod。

Mod loading API(應用編程接口)標準化的東西，允許Mod之間兼容。官方Minecraft API尚未發佈。在此期間，Minecraft開發者社區填補了這片空白：

  • ModLoader 是最舊的 API，在以前非常流行，在 Minecraft Forge 普及後它的受歡迎程度也漸漸消失了。
  • FML(Forge ModLoader)基本上只有Forge，但沒有Forge API而且只是mod加載器。FML也與ModLoader、ModLoaderMP、LiteLoader和M3L反向兼容並且用途相同。FML支持小mod，但是Forge還可以支持大的Mod。Lex Manos開發了FML和Forge。FML可以 在這裡下載。FML是開源的，其源代碼可以在此處查看。自從1.1開始，FML就已經分別對Forge發佈了 。modloader的mod用來作為java類，但是現在是.jar和.zip文件。FML可以在客戶端和服務端運作。
  • Minecraft Forge與ModLoader、ModLoaderMP、LiteLoader和M3L反向兼容，同時添加了成千上萬個的附加方法、類以及幫助Mod作者的實用工具。它增加了mods文件夾並且與安裝器一並產生。它對客戶端和服務端都有用。Forge已經被Risugami's ModLoader代替。Forge是和Forge API一起的FML。Forge自從Beta 1.7.3就有了。Forge和FML的製作者聞名於Lex Manos。Forge可以在此處下載。Forge是開源的，源代碼可以在此處查看 。Minecraft支持包、Forge文檔、ForgeGradle(也是開源的，代碼可以在此處查看)、Forge安裝器、mod列表輸出器(ModsList Outputter)和DiscourseForge都是開源的。Forge與損壞的Java 8版本20兼容，但對於Java版本20的兼容性問題已經在LegacyJavaFixerMod修復。modloader的mod用來作為java的類，但現在是.jar和.zip文件。從Beta 1.7.3到正式版1.12.1，Forge對這些版本都支持。舊版本的Forge可以在此處、此處和此處下載。LuaForge是用Python和Lua編寫並且也是開源的，開源代碼可以在此處查看，然而該項目不再維護，只保留歷史用途。
  • Bukkit是一個服務端流行的API。

反編譯 工具幫助你與 Minecraft 源代碼一起工作，不過這並不是由 Mojang 發佈的。

  • Minecraft Coder Pack 實用工具(MCP)是社區標準工具，用於反編譯和反混淆 Minecraft 代碼，讓代碼更容易修改和閱讀。
  • 通用 Java 反編譯器。例如 JD、JAD 和 fernflower，會幫助你在 MCP 未發佈或不可用(例如為快照編寫 Mod)時查看代碼。最好的一個可能是 CRF，因為它可能是唯一的支持最新 Java 和 JDK ，Java 8的 Java 反編譯器這裡是 Java 反編譯器列表。

一個 IDE (集成開發環境)對於任何不平凡的開發都是有用的。MCP 會自動生成 Eclipse IDE 的項目文件。當前最流行的 IDE 是 Eclipse，但是其它2個競爭對手是 NetBeans和IntelliJ(第三)。Xcode 比 IntelliJ 還要流行，但 Xcode 僅可用於蘋果電腦。

  • JBoss Developer 是一個大型的、非常好的 Eclipse 插件開發工具。i

把源代碼編譯成編程語言。你可以在 GitHub 和 BitBucket 看見你自己的 Mod 的源代碼以及其它 Mod 的源代碼。

  • Javac 是當下最流行的編譯器。但是 Power J 編譯器(第二)使用人數非常接近 Javac，曾經也都非常流行。緊隨其後的是 Jikes(第三)，ECJ(Eclipse Java 編譯器)(第四)和GCJ(GNU Java 編譯器)(第五)。
  • ECJ 是 Eclipse IDE 的一部分，Power J 是其它 IDE 的一部分

調試器用於搜索 Mod 代碼里的漏洞並幫助修復它們。

簡單 Mod 生成器，例如 Mod Maker 和 MCreator，專門為不想編寫和編譯 Java 代碼的 Mod 作者而制定。不推薦使用它們，因為它們創建的代碼質量很低，並且由很大的限制。

3D 建模和材質工具，例如 MCAnimator 或 Techne，幫助創建或修改一個實體或實體附加值。

原始記錄分析工具，許多這些工具都在 http://wiki.vg/Utility 上有描述，對多人遊戲 Mod 調試很有用。

其它 API 也是很有用的。示例：GuiAPI、LibShapeDraw、Player API，許多 1.7.10 Mod 的：核心、庫、工具或 API 都可以在這個 Mod 列表(英文)中找到。然而，一些 API 如IC2 API、Waila API、MattparksCore、Baubles API 和 Galacticraft API 沒被列出。

Minecraft Forge API

Minecraft Forge API 對開發 Mod 來說是非常有用的。它最重要的功能就是避免 Mod 之間的衝突。它也包含了開發 Mod 的便利方法，使開發 Mod 更容易；例如， addSmelting 方法只需要一行代碼就能添加燒煉配方。還有許多類似的方法。Forge API 教程隨處可見，這裡是 Forge 官方下載點。

小貼士

一個最簡單的mod類型就是增加一些新的模塊或物品。複製一個相似的類，並進行必要的修改。如果新的模塊或物品與已存在的物品分享了大部分的行為，那麼就可以保留那些相同的部分，而只修改少量的不同部分。根據所期望的它的行為，以及它將如何與其他的對象進行交互，或許有必要對其他的類進行修改，但那會增加與其他mod衝突的可能性。

不使用 MCP 來開發 Mod

有時候不用 MCP 開發 Mod 是很有用的。例如，MCP 不會發佈快照版。

為了不使用 MCP 開發 Mod，你必須反編譯、反混淆代碼或修改 Java 字節代碼。反編譯很容易，但是你只能反編譯一些簡單的類。
