區塊


區塊(Chunk)是Minecraft世界里一個大小為16×256×16的部分。

生成

區塊寬16格，長16格，高256格，共有65536格。當玩家第一次出現在世界時會在其周圍生成區塊，而隨著玩家對世界的探索，相鄰的區塊也會被生成。一個世界可生成14萬億(14,062,500,000,000)個真區塊。每個區塊有7.46*10244,700種可能的區塊，不包括實體。

在單人遊戲中所生成區塊的確切數字往往與視野距離和玩家的移動有關。而在多人遊戲里，在每位玩家周圍會生成一個以玩家為中心，半徑為10的二維區塊網格(一共21×21總計441個區塊)，儘管這可以被更改為3-15之間的一個值，通常只會在連接不良的家用服務器里調低此數值。在指定範圍內的區塊會有活動(生物生成、樹木生長、水流動、掉落的物品消失等)，而不在範圍內的區塊不會有交互，並會存儲在硬盤里(這個「按需加載」內存管理機制被廣泛用於程序化的地形生成，以便玩家的電腦不需要同時跟蹤並更新成百上千個植物和生物)。區塊每次存儲的時間間隔最小為30秒。

史萊姆可以只生成在特定區塊里，這些區塊由根據區塊坐標執行的計算而決定。有許多工具和Mod允許告知玩家可生成史萊姆的區塊，點擊此處查看詳情。

Minecraft的渲染器將一個區塊分為8個16x16x16的渲染區塊，並編譯到OpenGL顯示名單里用於加快渲染速率。任意一個渲染區塊內的方塊發生變動的話，渲染區塊均需要重建。

區塊是通過地圖種子進行創建的，這也意味著如果每次使用的地圖種子相同的話，所生成的區塊也會相同，前提是Minecraft和世界生成器的版本要一致。

出生點區塊

在出生點附近的區塊是一塊圍繞世界出生點的區域中的一個區塊，只要有玩家在主世界，它就不會被從內存中卸載。這意味著像紅石元件和刷怪會繼續，甚至所有玩家都距離它很遠時。


尋找區塊邊界

F3+G鍵可用來顯示區塊邊界。

按下F3鍵可打開調試屏幕以顯示玩家的X、Y和Z坐標以及「c」變量的值。這些坐標會因玩家的移動而改變。玩家可以通過在「x」和「z」變量旁邊的「c」變量得知他們所處的區塊。括號里的數字指示了玩家距離區塊西北角的距離，因此如果X後面的c是3(5)，Z後面的c是2(4)，玩家位於區塊(3,2)，從區塊西北角開始的方塊(5,4)里。

X和Z坐標除以16即為區塊之間的邊界。例如：(96，-32)即為一個四區塊的交點，X坐標在80到96區間內並且Z坐標在-48到-32區間內的方塊即為一個區塊，X坐標在96到112區間內並且Z坐標在-48到-32區間內的方塊即為另一個區塊，以此類推。當X和Z坐標為16的倍數時，此時玩家會穿越一個區塊。

本質上，當X和Z坐標可被16除盡時，玩家位於區塊的左上角(西北角)。

此外，玩家可根據此公式知道他們所處的區塊：
區塊的X坐標：Floor( X坐標 / 16 )
區塊的Z坐標：Floor( Z坐標 / 16 ) Floor是獲取小於等於指定數字的最大整數，如Floor( 27.9561 )是27。

換句話來說，如果X是27，Z是-15，區塊將會是( Floor( 27 / 16 ), Floor( -15 / 16 ) )區塊，這意味著玩家在區塊(1,-1)。

同時，玩家可以通過這個公式知道他距離西北角的距離：
(X 或 Z 坐標) AND 15

在攜帶版，當切換高品質的圖像品質時，世界會重新渲染，只加載玩家所處的區塊一瞬間。這個方法對尋找區塊邊界會很有用，因為在那一瞬間會顯示清晰的區塊邊界。

你知道嗎

如果玩家位於未生成的區塊里，世界會立即隱藏，直到玩家位於一個有效區塊里。如果玩家的Y坐標在區塊邊界外時，這就不會被觸發。
