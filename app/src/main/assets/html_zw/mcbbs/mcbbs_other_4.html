Scoreboard指令

scoreboard是一個很強大的指令(比execute還強大),不過我發現很多玩家都不懂...

所以我才想要做一個關於scoreboard的指令教程

要是你們還是不懂，那我就實在沒辦法了

1.objective/players

1.1.objective/add

作用：添加一個新目標

首先scoreboard的最基礎的指令:

/scoreboard objective add 目標名稱 判據 顯示名

這條指令會添加一個目標，名稱任意。這是scoreboard里最基礎的指令，沒有它scoreboard指令也就沒啥用了

你可以添加無限個目標，但名稱不能重復，判據可以重復

目標是什麼?

簡單來說，目標是一種計算分數的東西，所有目標都有這幾樣東西:

1.名稱,顯示名

2.玩家(或生物),分數

3.判據

判據是什麼?

每個目標都有它們所屬的判據，所有目標的判據可以相同，但名稱不能相同，如果出現了名稱相同的情況，

就會 .

一般來說，如果出現這種情況就會在名稱後面加幾個數字.判據有很多，判據使電腦知道這是一個什麼樣的目標，可以按Tab來查看:

(在下划線那裡按Tab鍵),然後彈出一個判據名稱，再按一次Tab就會再出

只要做了與判據相符的事,就會加一分

下面是一個例子:

/scoreboard objective add Test dummy

這個指令運行後會添加一個名為"Test"的變量

如果開啓了commandBlockOutPut,左下角的顯示是正常的，你就己經添加了Test目標

當然判據可以變:

/scoreboard objective add Test1 deathCount

這樣就會添加一個判據為deathCount的新目標"Test1"

有權限的玩家可以改變"Test"或"Test1"的分數(如果判據不是dummy也可以改,只不過判據也可以自動加分)

1.2.players/add

作用：給玩家的某目標加分

如果只是單純地添加一個目標，而不對它做任何有意義的事情，那這又有什麼用呢???

所以說目標還是很有用處的，比如說他可以給特指的玩家（或生物）加分

加分:

/scoreboard players add 選擇器 目標名稱 分數

這條指令可以讓玩家修改玩家在某個目標的分數

比如:

/scoreboard players add @p Test 1

這條指令可以讓最近玩家的目標分數(Test)加1分

分數你可以改成任何數字

當然你不止可以加分數，還可以設置分數(set),和減分數(remove):

1.3.players/set/remove

作用：給玩家的某目標設置分數和減去分數

設置分數和減去分數:

/scoreboard players set 選擇器 目標名稱 分數

/scoreboard players remove 選擇器 目標名稱 分數

比如: /scoreboard players set @p Test 0

/scoreboard players remove @p Test 1

這兩條指令分表讓最近玩家的目標分數(Test)設成0和減1

將玩家在某目標的分數顯示：

如果你想讓分數顯示在左邊,顯示位置就填sidebar

1.5.檢測score

作用：檢測玩家的分數

選擇器[score_目標名稱_min=檢測分數]

選擇器[score_目標名稱=檢測分數]

問:這是什麼鬼東西?

這是用檢測玩家分數並做出舉動,比如:

/tellraw @a[score_Test_min=20] [{"text":"你贏了"}]

這條指令的意思是:

告訴目標為Test的玩家的分數大於等於20時，就會打出"你贏了"

如果是@a[score_Test=20]就是小於等於20當然你可以兩個一起用，比如:@a[score_Test_min=20,score_Test=20],意思就是等於20時就會...

這樣就可以起到一個檢測作用,不過要保持開啓

當然，如果這樣的話就會一直顯示"你贏了"

所以後面一個命令方塊還得用來重置:

/scoreboard players set @a[score_Test_min=20] Test 0

這樣既能顯示一個你贏了,同時重置了分數

如果你手滑想要刪除一個目標時:（下面）

移除一個目標：

1.8.players/operation

作用：在目標里做運算

在我的世界做運算，可以用到operation

/scoreboard players operation 主要針對對象 主要針對對象目標名稱 操作符 次要針對對象 次要針對對象目標名稱

這看起來有點難理解，解釋一下:

假如A玩家的目標名為score1的分數為2，B玩家的目標名為score2的分數為1,再用此指令:

/scoreboard players operation A玩家 score1 += B玩家 score2

這樣A玩家目標名為score1的分數就變成了3(2+1)

主要針對對象和目標和次要針對對象和目標名稱可以填同一個

當然操作符不止一個,列一個表:

操作符 意思
+=加法，只改變主要針對對象的目標分數，不改變次要針對對象分數
-=減法，只改變主要針對對象的目標分數，不改變次要針對對象分數
*=乘法，只改變主要針對對象的目標分數，不改變次要針對對象分數
/=除法，整除，只改變主要針對對象的目標分數，不改變次要針對對象分數
%=除法，余除，只改變主要針對對象的目標分數，不改變次要針對對象分數
<當主要針對對象分數<次要針對對象分數，兩個都不變
當主要針對對象分數>次要針對對象分數，次要針對對象的分數會賦值給主要針對對象的分數，次要針對對象的分數不變
>當主要針對對象分數>次要針對對象分數，兩個都不變
當主要針對對象分數<次要針對對象分數，次要針對對象的分數會賦值給主要針對對象的分數，次要針對對象的分數不變
=直接將次要針對對象的分數賦值給主要針對對象分數
><直接將兩個分數賦值給對方

1.9.players/reset

作用：將玩家的目標重置

將玩家的特顯欄清除，並把玩家的分數重置為0：

1.12.腦補/計時器

現在你們應該已經學會了一點scoreboard指令,現在就給大家介紹一個用scoreboard指令做的計時裝置,給大家腦補腦補

有三個指令方塊:

這裡給大家介紹的是倒計時（remove）,當然大家也可以加分計時(add)

首先得要加兩個變量:

/scoreboard objective add time1 dummy

/scoreboard objective add time2 dummy

圖片中從下往上的指令分別是(循環):

/scoreboard players add @a time2 1 給目標名為time2的所有玩家加一分

/scoreboard players remove @a[score_time2_min=20,score_time1_min=1] time1 1

如果目標time2大於等於20,time1也大於等於1，就給time1減一分(這裡也可以設置成add)

/scoreboard players set @a[score_time2_min=20] time2 0

如果目標名為time2的分數大於等於20,就把time2設成0

這可能對萌新看不懂，解釋一下:

首先time2是用來輔助time1計時的,time2一直在給自己加分，當加到了20分（也就過了一秒），就把time1減掉一分並把time2重新設置成0，這樣就起到了一個循環作用

time1一開始你要設置分數(時間)，當time1減到0時，就不會再減下去了，因為第二條指令中的score_time1_min=1],0小於1，所以就不減了

當然你也可以用setdisplay來顯示計時的時間,讓玩家知道還有多少時間

你也可以正計時,把第二條指令改成這樣:

/scoreboard players add @a[score_time2_min=20,score_time1=分數] time1 1

這樣就形成了正計時

分數要改成你要計時的分數,比如499(到了500就不加了,剛剛好),99,149

2.teams

記分板是給每個人用的，但隊伍是給個別人用的，不同的隊伍可以設置不同的東西

比如可以創建紅隊，綠隊:

2.1.teams/add

作用：添加一個隊伍

/scoreboard teams add 隊伍名稱 顯示名

這樣你就可以創建兩個不同的隊伍,比如:

/scoreboard teams add Red Red

/scoreboard teams add Green Green

這樣就創建了兩個隊伍:Red,Green

但是雖然創建了兩個名字相同的隊伍，但裡面都沒人，所以要加點人進去:

2.2.teams/join

作用：讓玩家加入某隊伍

/scoreboard teams join 隊伍名稱 選擇器

這樣就可以添加點人進去，比如:

/scoreboard teams join Red @p

/scoreboard teams join Green @p

如果一個人先前加入了隊伍Red,後來又加入了隊伍Green,那麼這個人會自動退出隊伍Red並自動加入隊伍Green

2.3.teams/option

作用：給某隊伍設置選項

雖然加入了隊伍紅、綠，但還沒設置它們的顏色

/scoreboard teams option 隊伍名稱 設置 設置效果

我們現在要改變顏色，所以:

/scoreboard teams option Red color red

/scoreboard teams option Green color green

前面的Red指的是隊伍名稱，後面的red指的是顏色

option後面可以跟很多東西，比如friendlyfire、collisionRule

/scoreboard teams option Red friendlyfire false

這樣就隊伍Red中的人就打不到對方

下面是把碰撞體積關掉

/scoreboard teams option Red collisionRule never

option選項有很多，可以按Tab查看更多
