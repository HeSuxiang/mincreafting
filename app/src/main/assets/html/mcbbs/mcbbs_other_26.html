Main

我们先来复习一下上次我们讲的主方法里面有什么。

public static void main(String[] args)

公共的 静态的 空的 主要的（字符串[] 参数）

我们还可以看到这个main方法里面有一个 // TODO Auto-generated method stub，翻译过来是“// TODO自动生成的方法存根”，

也就是我们可以使用 //你想要注释的内容 给类写备注。

一般我们会在备注里面说明某段代码的具体作用，或者使用//将可能错误的代码暂时注释掉，方便debug（调试）。

//为单行注释，也就是说当前行的内容不会被当成执行语句执行。

/**/为多行注释，在此范围内的多行备注不会被执行。

/***/为doc注释，在此范围内的多行备注不会被执行、给其他使用你代码的人提供了一个可以参阅的doc文档。

控制台打印输出

接下来我们在main方法里面输入：

System.out.print("");

然后在""内输入任意字符。（记得双引号一定是半角）

点击 左上角绿三角图标运行。（如果你没保存会提示保存）

我们就会在控制台看到输出的语句：

刚刚输入的代码是什么意思呢？

翻译过来就是“系统.输出.打印”。

这是一条在控制台打印语句的代码，方便我们进行调试。

那么如果我想输出第二行换行的语句呢？

将这段话复制并不会自动换行，那么我们需要在第一条语句后面加上ln。

System.out.println("");是一个输出完后换行的语句，同样的我们在""内输入任意字符即可。

看，是不是换行了~

加上ln代表在这条语句结束后换行。

当然你也可以将""删除，在()内填入数字、数学运算式，同样可以运行~

尝试运行后我们发现控制台显示了数字，数学运算式结果。

那么为什么数字是直接显示数字，而数学运算式显示的是结果呢？

因为之前带有""的显示为文本，不进行计算。

而去掉""后，如果()里面的是运算式，将进行计算。

在这里我们将 ""这种 文本形式称为String字符串类型。

1、2、3...这种 整数的数字类型称为int整型。

那小数怎么办呢？当然也有小数的类型，不过暂时不用，所以先不讲解。

那么MC里有没有一个显示文字的方式呢？

RunClient（HelloMinecraft！）

小伙伴们可能有个疑问，如果每次构建模组的时候都要把模组丢到客户端里试，那不是很麻烦？

其实下载的mdk可以通过cmd来直接运行客户端。也就是说这个mdk包含客户端启动需要的东西。

在构建完毕后将cmd切换到mdk目录下，运行指令gradlew.bat runClient 就可以运行客户端。

之后如果运行过gradlew.bat build构建好mod，运行的客户端里面会包含已经构建好的mod。

可以在build\lib找到构建好的mod~

听说可以直接点eclipse里面的运行来运行客户端...渣渣的我不会。

在制作mod前我们还需要配置一个东西，让客户端识别jar文件是mod。
