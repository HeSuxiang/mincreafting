环境变量与配置

①打开我的电脑->控制面板->高级系统设置

②系统属性->高级->环境变量设置

③可以看到新建变量、编辑变量、删除变量的选项

新建系统变量，

变量名JAVA_HOME，

变量值为安装的jdk路径。

例：C:\Program Files (x86)\Java\jdk1.8.0_91

在变量Path增加一句;%JAVA_HOME%\bin;

注意前面和后面是英文分号

新建系统变量，

变量名CLASSPATH，

变量值为已安装下jdk\lib\tools.jar，和已安装jdk下的\lib\dt.jar。

你可以填写jdk的绝对路径，

例：D:\JDK\lib\dt.jar，

也可以填写JAVA_HOME，例：%JAVA_HOME%\lib\dt.jar。

*记得用分号;将新的变量值和旧的变量值隔开。

*感谢16L 疑问的化名 补充：Win10尽量的Path变量尽量直接指向路径，否则视win10版本可能会产生bug

eclipse工作环境配置

打开一次eclipse，设置工作环境位置(workspace)。

解压mdk成文件夹，放入Eclipse的workspace下。（建议小伙伴改成方便输入的文件名）

还记得mdk里面都有什么吗？

build 里存放我们构建好的mod。

src 里存放我们的源代码。

gradlew.bat 用它来构建我们的mod！

CMD设置工作环境：

CMD是什么？CMD就是这个黑黑的东西。

你可以通过开始->运行->输入cmd找到它

打开后它是这个样子：


黑不溜秋的就像N年前的电脑一样...

接下来我们要学两个简单的CMD命令，一个是切换盘符，一个是进入文件夹。

【切换盘符】直接输入D:即可切换到D盘下，其他盘同理。

【进入文件夹】cd 文件夹名 或 cd 文件夹名\文件夹名\文件夹名

譬如我的workspace在d盘下，那么输入d：后回车进入d盘，

然后输入 cd workspace 进入工作环境下，

再输入 cd mdk文件夹名进入mdk包中。

使用命令gradlew.bat setupdecompworkspace 设置工作环境。

出现Build Successful为构建成功。

Q&A卡住了怎么办？
