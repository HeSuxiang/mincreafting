代码是如何被执行的呢？

我们写的类文件以.java为后缀，这时计算 机并不能识别我们写出的代码，因为我们的代码是给人看的。

那么

想要计算机读懂我们的语言，我们就需要一个人进行翻译，这个人

就是编译器。

经过编译器进行翻译后生成了以.class为后缀的文件，

计算机(java虚拟机)可以识别，然后计算机识别class文件来执行。

写好代码>编译成Class>执行

我们学习了一个普通的java项目，需要一个带有main的主类才能运行，那么在mod中我们同样需要一个主类来运行。

现在我们来了解一下一个mod主类的基本构造：

@Mod

带有@符号的表示这个是一个注解。（跟注释不一样的！）

注解用来标识下方的方法或类，告诉编译器这是一个怎样的东西。

譬如在mod开发中我们在一个类上方使用@Mod，表示告诉了编译器这是一个mod主类。

那么我们来开始操作吧，首先在src下创建包，再创建一个普通类（也就是不需要勾选main）

然后在类名上方使用@Mod注解：

看到这里我们知道，我们可能缺少了什么东西，

导致软件不能识别我们的@Mod（因为这个注解是MC里面的）。

点击import Mod导入MC所需要的包。

导入完毕后还是会红线报错，这时我们将

(modid = 类名.MODID, name = 类名.NAME, version = 类名.VERSION, acceptedMinecraftVersions = "1.7.10")

这段代码复制到@Mod后面，告诉@Mod我们已经按它的格式写好了modid、名字、版本号、可运行的MC版本。
