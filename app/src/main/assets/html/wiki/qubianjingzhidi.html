去边境之地

边境之地已经在Beta 1.8修复了。然而，它仍在携带版中存在,且你可以把这样的地形传到更新的版本中.


边境之地。

边境之地是一个非常危险的区域，不仅仅是对玩家危险（无尽的黑暗，敌对生物和必须小心坠落），还对运行着Minecraft的计算机有危险。但是，如果你真的想要去那里，这个页面可以告诉你如何去并且如何减少卡顿。

如何去

边境之地非常非常遥远，它们从X/Y轴±12,550,821的位置才开始在任意轴生成。但即使这样，也有很多方法来达到边境之地。

注：用斜体标注的方法无法在携带版实现。

步行

步行是几种方法里面花费最长时间的一种，从世界中心到边境之地需要一个月以上的 Minecraft 游戏时间。 然而，你也可以用下面的方法。

下界传送门

在下界走过一个方块相当于在主世界走过8个方块。因此，通过下界很可能只要一周的时间到达主世界的边境之地。

最后一个传送门的最最好位置是在下界任意一个轴上的±1,568,852之前。

这个方法只能用于到达主世界的边境之地。

使用命令

使用 命令 , 它使玩家去到边境之地成为可能。使用命令 / tp 或在命令方块中输入：

  • /tp <玩家> 0 130 12550800 或 /tp <玩家> 12550800 130 0 并激活命令方块通往边境之地。确保已开启飞行并且玩家伤害已关闭，因为它会传送到高度限制之外。0可以被 -12,550,820 到 12,550,820 之间的数字代替。
  • /tp <玩家> 12550800 130 12550800 传送到边境之角附近。同时你要确保你不会受到跌落伤害。

上述两种情况的 12550800 可以是负的。

在下界中，这些命令会把你传送到顶端的 基岩 , 所以一定要确保高度很低。 打开 "no-clip" ，使用旁观者模式 。如果可能，则可以同时使用。

避免卡顿

由于溢出的坐标和实体，到达边境之地后游戏会发生严重卡顿。这可以通过命令来缓解。 如果支持命令，可以使用/kill @e[type=!Player] 来移除其他的实体（每分钟3次左右即可），而/gamerule doEntityDrops false 命令可以关闭物品掉落。 如果不支持命令，可以通过Mod来实现类似效果。

将边境之地地形移植到新版本

我们可以将边境之地的地形从Beta 1.8版本移植到更高的版本中。在Beta 1.8版本到达边境之地后，继续前进让游戏继续生成边境之地地形的区块。 而后将存档复制到新版本游戏中，打开存档并让其自动转换存档格式即可。
